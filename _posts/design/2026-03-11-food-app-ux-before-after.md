## 문제점 분석

기존 앱에서 발견된 UX 문제점을 정리했습니다.

### 1. 헤더 공간 낭비
- 헤더가 불필요하게 크고 강렬한 빨간색으로, **콘텐츠 영역을 줄이고** 시각적 피로감을 줍니다.
- 사용자에게 전달하는 정보 없이 공간만 차지합니다.

### 2. 카테고리 버튼 터치 영역 부족
- 버튼이 **너무 작고(padding 4px)** 간격도 좁아 손가락으로 정확히 누르기 어렵습니다.
- 모바일 터치 타겟은 최소 **44px × 44px**이 권장됩니다 (Apple HIG 기준).

### 3. 정보 계층(Hierarchy) 부재
- 메뉴 이름, 가격, 설명이 **모두 동일한 크기와 색상**으로 되어 있어, 어떤 정보가 중요한지 한눈에 파악하기 어렵습니다.
- 카드 레이아웃 없이 단순 나열 → 메뉴 간 구분이 약합니다.

### 4. CTA(Call to Action) 버튼이 약함
- "담기" 버튼이 회색 테두리에 작은 글씨로, **행동 유도력이 거의 없습니다**.
- 사용자가 어떤 행동을 해야 하는지 시각적 신호가 부족합니다.

### 5. 장바구니 피드백 없음
- 장바구니 플로팅 버튼이 **회색으로 묻혀서** 존재감이 없습니다.
- 담기 버튼을 눌러도 **아무런 피드백이 없어** 실제로 담겼는지 알 수 없습니다.

### 6. 상세 페이지 CTA도 약함
- 상세 페이지의 "장바구니에 담기" 버튼도 회색 테두리 스타일이라 **주요 액션 버튼으로 인지하기 어렵습니다**.

## 개선 포인트 요약

| 문제 | Before | After |
|------|--------|-------|
| 헤더 | 크고 강렬한 빨간 배경 | 컴팩트 + 인사말로 친근감 |
| 카테고리 | 4px 패딩, 아이콘 없음 | 8px+14px 패딩, 이모지 아이콘 추가 |
| 카드 레이아웃 | 단순 나열, 구분선 없음 | 가로 카드형 + 둥근 모서리 + 그림자 |
| 타이포 위계 | 이름·가격 동일 스타일 | 이름(굵게), 가격(컬러 강조), 설명(회색) |
| CTA 버튼 | 회색 테두리, 10px 글씨 | 주황 배경, 흰 글씨, 둥근 Pill형 |
| 장바구니 | 회색 36px 원 | 주황 52px + 그림자 + 뱃지 카운트 |
| 피드백 | 없음 | 토스트 메시지 + 뱃지 숫자 증가 |

## 핵심 교훈

### 작은 변화가 큰 차이를 만든다

이번 개선에서 **기능은 하나도 바뀌지 않았습니다**. 바뀐 것은:

- **여백과 크기** — 터치 영역, 카드 간격, 버튼 패딩
- **색상과 위계** — 중요한 정보에 시선이 가도록 컬러 강조
- **피드백** — 행동에 대한 즉각적 반응 (토스트, 뱃지)
- **친근함** — "맛있는 음식 주문" → "오늘 뭐 먹을까요?"

결국 UX 개선의 본질은 **"사용자가 생각하지 않아도 되게 만드는 것"**입니다.

> "Don't make me think." — Steve Krug

다음 포스팅에서는 이 앱의 **사용성 테스트 결과**와 실제 사용자 반응을 분석해 보겠습니다.
