## UX 테스트란?

UX 테스트(Usability Testing)는 실제 사용자가 앱을 사용하는 과정을 관찰하고, 사용성 문제를 발견하는 디자인 검증 방법입니다. 디자이너의 의도대로 사용자가 앱을 사용하는지, 어디서 막히는지를 직접 확인할 수 있습니다.

## 왜 UX 테스트가 필요한가?

디자이너는 앱의 구조와 흐름을 너무 잘 알고 있기 때문에, 사용자가 겪는 어려움을 놓치기 쉽습니다. UX 테스트를 통해 다음을 확인할 수 있습니다:

- **사용자가 핵심 기능을 쉽게 찾는가?**
- **화면 간 이동이 자연스러운가?**
- **버튼, 텍스트, 아이콘이 직관적으로 이해되는가?**
- **작업 완료까지 몇 단계가 걸리는가?**

## UX 테스트 진행 방법

### 1. 목표 설정

테스트 전에 검증하고 싶은 항목을 명확히 정합니다.

> 예시: "사용자가 회원가입 후 첫 번째 프로필 설정을 3분 이내에 완료할 수 있는가?"

### 2. 테스트 시나리오 작성

사용자에게 수행할 과제(Task)를 구체적으로 작성합니다.

| 과제 | 설명 |
|------|------|
| Task 1 | 앱을 열고 회원가입을 완료하세요 |
| Task 2 | 프로필 사진을 변경하세요 |
| Task 3 | 설정에서 알림을 끄세요 |

아래는 이번 UX 테스트에 사용한 앱 화면 샘플입니다:

<div class="phone-mockup">
  <div class="phone-frame">
    <iframe src="/assets/html/phone/ux-test-sample.html"></iframe>
  </div>
</div>

### 3. 참여자 모집

- 실제 타겟 사용자 **5명** 정도면 주요 사용성 문제의 약 80%를 발견할 수 있습니다 (Jakob Nielsen 연구 기반).
- 앱에 익숙하지 않은 사람을 섭외하는 것이 중요합니다.

### 4. 테스트 진행

- **Think Aloud (소리 내어 생각하기)**: 사용자가 앱을 사용하면서 생각을 말로 표현하도록 합니다.
- 디자이너는 **관찰자** 역할만 하고, 힌트를 주지 않습니다.
- 화면을 녹화하여 나중에 분석합니다.

### 5. 결과 분석 및 개선

테스트 결과를 정리하고, 문제의 심각도를 분류합니다.

| 심각도 | 설명 | 예시 |
|--------|------|------|
| 🔴 Critical | 과제 수행 자체가 불가능 | 버튼이 반응하지 않음 |
| 🟠 Major | 과제 수행에 큰 어려움 | 메뉴 위치를 찾지 못하고 헤맴 |
| 🟡 Minor | 불편하지만 수행 가능 | 텍스트가 작아 읽기 어려움 |

## 도구 추천

- **Figma Prototype** — 코딩 없이 인터랙티브 프로토타입 테스트 가능
- **Maze** — Figma와 연동하여 원격 UX 테스트 진행
- **Lookback** — 사용자 화면 및 표정 녹화

## 마무리

UX 테스트는 거창한 리서치가 아닙니다. 프로토타입 하나와 사용자 5명만 있으면 디자인의 방향이 맞는지 빠르게 검증할 수 있습니다. 디자인을 만드는 것만큼, **검증하는 과정**이 좋은 앱을 만드는 핵심입니다.
